# Diarios
Puedes registrar tus reflexiones durante el proceso de codificación de datos mediante los diarios. Utilízalos para desarrollar ideas y ejes temáticos. 

<img width="895" height="706" alt="image" src="https://github.com/user-attachments/assets/016f7e1d-0970-401a-bbb1-6526687d5fc9" />

Los diarios pueden exportarse de forma individual o, si lo prefieres, puedes recopilar y exportar el texto de todos los diarios en un solo archivo.

Los diarios se pueden eliminar y también pueden contener atributos.

Existe una función de búsqueda para localizar información dentro del texto de los diarios. Esta herramienta emplea símbolos de expresiones regulares (_regex_) para ofrecerte mayor flexibilidad.

* El punto '.' funciona como un comodín; por ejemplo, .atas coincidirá con "patas" y "ratas".
* El signo '?' después de un carácter buscará cero o una ocurrencia de dicho carácter; por ejemplo, gatos? coincidirá tanto con "gato" como con "gatos".
* Para buscar los símbolos literales, utiliza la barra invertida: "\." para el punto, "\?" para el signo de interrogación y "\n" para representar un salto de línea.

Guía rápida de regex: 
www.rexegg.com/regex-quickstart.html

Puedes ordenar los diarios por nombre o por fecha, así como añadirles atributos. Mediante el menú contextual (clic derecho), tienes la opción de mostrar únicamente las filas de los diarios que coincidan con un nombre específico, un codificador o un valor de atributo.

La ventana del diario es flotante, lo que te permite redactar notas mientras realizas la codificación o generas informes.

## Resaltar texto del diario mediante Markdown

Tienes a tu disposición algunas funciones de Markdown para resaltar el contenido de tus diarios.
Puedes utilizar de uno a tres símbolos de almohadilla (#) para definir niveles de encabezado. Asimismo, emplea un solo asterisco (*) rodeando las palabras para obtener texto en _itálicas_ y doble asterisco (**) para resaltar texto en **negritas**.
