# Creación de un nuevo proyecto

Para crear un nuevo proyecto (Ver imagen 1), dirígete al menú "Proyecto". Una vez que se despliegue el menú, selecciona "Crear nuevo proyecto" o utiliza el atajo **Ctrl+N**. Esto abrirá una ventana del explorador de archivos. En esta nueva ventana, escribe el nombre que deseas darle a tu proyecto en el campo de texto ubicado en la parte inferior. Finalmente, para completar la creación del proyecto, haz clic en el botón "Guardar".

Si lo nombras "Investigación", el proyecto se guardará como **Investigación.qda**. En realidad, **Investigación.qda** es una carpeta con subcarpetas que contienen la base de datos, los documentos empíricos y otros archivos (Para más información consulta la [Estructura de datos](https://qualcoder-org.github.io/doc/es/7.3.-QualCoder-Development/)). Si deseas enviar el proyecto por correo electrónico a alguien más, debes comprimir la carpeta completa en formato **.zip**. 

## Pantalla principal

La pantalla principal de _QualCoder_ se organiza mediante una barra de menús superior y cinco pestañas de acceso rápido: Registro de acciones, Administrar, Codificación, Informes y Chat IA (Ver imagen 2). La pestaña "Registro de acciones" (Action Log) muestra los diversos cambios que se realizan al utilizar _QualCoder_. Por lo general, es seguro cambiar directamente entre pestañas. 

**Imagen 2:** Menús y pestañas de acceso rápido

<img width="891" height="621" alt="image" src="https://github.com/user-attachments/assets/3a516b88-b1d9-447f-94c6-fc4c3a0c30ce" />

Sin embargo, si estás añadiendo, renombrando o eliminando archivos en "Administrar archivos", o cambiando códigos y categorías en el área de codificación, es preferible actualizar cada pestaña seleccionando la opción correspondiente en la barra de menú superior. Por ejemplo, si realizas cambios en la administración de archivos y luego procedes a codificar o generar informes, selecciona la opción en el menú de "Codificación" o "Informes" en lugar de hacer clic directamente en la pestaña. Esto se debe a que es posible que algunos cambios de gestión no se actualicen automáticamente.

**Importante: No ejecutes tu proyecto desde una ubicación en la nube** (OneDrive, Dropbox, etc.). Las razones son: en ocasiones los archivos no pueden importarse al proyecto; además, acceder a un proyecto almacenado en la nube desde diferentes computadoras es especialmente riesgoso, ya que puede causar problemas de sincronización e incluso corromper la base de datos. Consulta [esta página para conocer estrategias seguras de trabajo en equipo y en múltiples computadoras](https://qualcoder-org.github.io/doc/es/2.4.-Working-in-a-Team). De igual manera, **no ejecutes tu proyecto desde una memoria USB**, ya que esto puede provocar la corrupción de la base de datos.

**Buenas prácticas de gestión de datos**. Para cualquier proyecto, mantén buenas prácticas de gestión de datos, como realizar respaldos de forma regular y, de ser posible, almacenarlos en diferentes ubicaciones (por ejemplo, en la nube, en distintas carpetas de la computadora, discos duros o memorias USB).

## Redactar un memo del proyecto

Esto es **especialmente importante si deseas utilizar las funciones basadas en IA** en _QualCoder_. El memo del proyecto se utiliza para proporcionar información contextual sobre el proyecto actual y el tipo de documentos empíricos con los que se está trabajando. Esta información, que se incluye en cada consulta, ayuda a la IA a generar respuestas mucho más precisas y útiles. 

Puedes acceder al **Memo del proyecto** de tres maneras distintas:

**1. Atajo de teclado:** Presionando simultáneamente las teclas **CTRL+M**.

**2. Menú Proyecto:** Seleccionando la opción "Memo de proyecto" dentro de la pestaña desplegable (Ver imagen 3).

_QualCoder_ proporciona una plantilla para la información más relevante. No es obligatorio seguir esta plantilla al pie de la letra, pero intenta **mantener la descripción del proyecto breve** (entre 300 y 700 caracteres). También puedes añadir notas sobre tu marco teórico, pero ten cuidado de no "preformatear" demasiado la perspectiva de la IA. 

Si también deseas utilizar el memo del proyecto para otras notas que no deban compartirse con la IA, puedes insertar la etiqueta ##### en el texto. Cualquier contenido debajo de esta marca no se enviará a la IA (esto funciona para cualquier memo en _QualCoder_). 

Si prefieres que el memo del proyecto no se envíe a la IA bajo ninguna circunstancia, puedes desactivar esta opción en la ventana de "Configuraciones".
