# Instructions d'installation et d'exécution de QualCoder

## INSTALLATION

Pour les installations manuelles (via la ligne de commande ou le terminal), je mentionne une version spécifique, par exemple la version 3.6. Veuillez vérifier sur la [page des versions](https://github.com/ccbogel/QualCoder/releases) pour obtenir la version la plus récente, qui peut être plus avancée.

### Prérequis
VLC est optionnel, mais nécessaire si vous souhaitez effectuer du codage audio/vidéo. Optionnellement, installez ffmpeg pour la conversion parole-texte et l'affichage des formes d'onde, voir [phoenixnap.com/kb/ffmpeg-windows](https://phoenixnap.com/kb/ffmpeg-windows).

### Windows

Consultez [https://github.com/ccbogel/QualCoder/releases/3.7](https://github.com/ccbogel/QualCoder/releases/3.7)

Vous avez deux options pour les exécutables, créés sous Windows 11 et pouvant fonctionner sous Windows 10 :

- Un installateur Windows est disponible sur la page des versions : **QualCoder.Win.3.7.exe**

- Une version portable, non installable, est disponible sous forme de dossier zip : **QualCoder.Win.3.7.Portable.zip**. Téléchargez et décompressez-le. Le fichier exécutable se trouve dans un dossier, accompagné d'un sous-dossier interne. Ces deux éléments doivent rester ensemble pour que l'exécutable fonctionne. Double-cliquez sur l'exécutable QualCoder pour le lancer (le démarrage peut prendre jusqu'à 20 secondes).

### macOS

**Installation via les bundles d'application :**

Assurez-vous d'avoir installé toutes les mises à jour macOS avant de commencer. Allez dans l'application Réglages système de votre Mac. Cliquez sur Général dans la barre latérale (il peut être nécessaire de faire défiler), puis sur Mise à jour logicielle.

Pour installer QualCoder :
Double-cliquez sur le fichier .dmg téléchargé. Glissez QualCoder vers le lien de votre dossier Applications.

Lancez QualCoder depuis le Launchpad ou en double-cliquant sur l'application dans votre dossier Applications.
Nous ne pouvons pas actuellement signer les bundles d'application. Lors du premier lancement, un message peut apparaître indiquant que QualCoder provient d'un développeur non enregistré ou que "QualCoder ne peut pas être ouvert car Apple ne peut pas vérifier l'absence de logiciels malveillants". Selon votre processeur, vous verrez l'une de ces options :

- Si vous voyez "Afficher dans le Finder" ou OK, cliquez sur OK.
- Si vous voyez "Terminé" ou "Placer dans la corbeille", cliquez sur "Terminé".

Ensuite, allez dans Réglages > Confidentialité et sécurité. Faites défiler jusqu'à voir un message indiquant que QualCoder a été empêché de s'ouvrir. Cliquez sur "Ouvrir quand même". Vous devrez peut-être entrer votre mot de passe et cliquer à nouveau sur "Ouvrir". Après avoir entré votre mot de passe, attendez quelques secondes que le programme se lance.

Désormais, QualCoder devrait démarrer sans problème.

### Fedora 43

Ces instructions ont été testées avec QualCoder 3.7 sur Fedora 43 avec Python 3.12.12.

Installez les dépendances :

`sudo dnf install python3.12`

Configurez QualCoder :

`cd ~/qualcoder  # remplacez par l'emplacement approprié sur votre machine`

`python3.12 -m venv env`

`source env/bin/activate`

`python3.12 -m ensurepip`

`python3.12 -m pip install --upgrade pip`

`mkdir tmp`

`TMPDIR=./tmp python3.12 -m pip install -r requirements.txt`

`deactivate`

Utilisation :

`cd ~/qualcoder  # remplacez par l'emplacement approprié sur votre machine`

`source env/bin/activate`

`cd src`

`python3.12 -m qualcoder`

### Fedora 42 

Ces instructions permettent de télécharger le code source actuel directement depuis GitHub. Remarque : Fedora utilise Wayland, qui peut ne pas fonctionner correctement avec l'interface graphique Qt. Il est recommandé d'installer également Xwayland.

Téléchargez le code source. Lisez et appliquez la section relative à l'utilisation du code source Fedora dans le fichier **Readme.md**.
Supprimez les guillemets inversés qui entourent les instructions de la ligne de commande `, car ils ne servent qu'à l'affichage.

QualCoder peut rencontrer des problèmes avec l'audio/vidéo sous Fedora.
