# Créer un nouveau projet
Vous pouvez **créer un nouveau projet depuis le menu « Projet »**.

Si vous le nommez « Test », le projet sera enregistré sous le nom Test.qda. Test.qda est en réalité un dossier contenant des sous-dossiers avec la base de données, les documents empiriques et d'autres fichiers. Si vous souhaitez envoyer le projet par email à quelqu'un d'autre, compressez l'intégralité du dossier.

---

## Écran principal

L'onglet **Journal des actions** dans la fenêtre principale affiche les différentes modifications apportées lors de l'utilisation de QualCoder. La fenêtre principale comporte une barre de menu en haut qui regorge d'options. Elle comporte cinq onglets : _journal des actions_, _gestion_, _rapports_, _codage_ et _chat IA_. Il est généralement possible de passer directement d'un onglet à l'autre. Mais si vous ajoutez, renommez ou supprimez des fichiers dans « Gérer les fichiers » (Gérer les fichiers) ou modifiez des codes ou des catégories de codes dans la zone de codage, il est préférable d'actualiser chaque onglet en sélectionnant l'option correspondante dans la barre de menu en haut de l'écran. Par exemple, si vous modifiez des fichiers dans « Gérer les fichiers », puis effectuez un codage ou créez des rapports, sélectionnez une option dans le menu pour le codage ou le rapport, plutôt que de cliquer directement sur l'onglet correspondant. En effet, certaines modifications de gestion peuvent ne pas être actualisées.

![image](https://github.com/user-attachments/assets/847f5523-5b5c-4a89-b116-3d3e54812558)

Il est **déconseillé d'exécuter votre projet depuis un emplacement cloud** (OneDrive, Dropbox, etc.) et de l'utiliser directement depuis là. En effet, certains fichiers peuvent ne pas s'importer correctement dans le projet. Accéder à un projet stocké dans le cloud depuis différents ordinateurs est particulièrement dangereux, car cela peut provoquer des problèmes de synchronisation et même corrompre la base de données. Consultez [cette page pour des stratégies sûres de travail en équipe et sur plusieurs ordinateurs](https://qualcoder-org.github.io/doc/fr/2.4.-Working-in-a-Team). De même, **n'exécutez pas votre projet depuis une clé USB**, car cela peut entraîner une corruption de la base de données.

Bonnes pratiques de gestion des données : pour tout projet, maintenez de bonnes pratiques de gestion des données, comme des sauvegardes régulières, idéalement stockées à différents endroits (cloud, dossiers sur différents ordinateurs, disques durs, clés USB).

## Rédiger un mémo de projet

Cela est **particulièrement important si vous souhaitez utiliser les fonctionnalités basées sur l'IA** dans QualCoder. Le mémo de projet permet de fournir des informations contextuelles sur le projet en cours et le type de documents empiriques traités. Ces informations, incluses dans chaque requête, aident l'IA à produire des réponses beaucoup plus ciblées et utiles. Vous pouvez accéder au **Mémo de projet** via le menu **Projet > Mémo de projet**.

<img src="https://github.com/user-attachments/assets/c4c44cbe-9195-4cd3-a1f6-e9efccceb45d" width=500em alt="Exemple de capture d'écran du mémo de projet">

QualCoder propose un modèle pour les informations les plus importantes. Vous n'êtes pas obligé de suivre ce modèle à la lettre, mais essayez de **garder la description du projet courte** (entre 300 et 700 caractères). Vous pouvez également ajouter des notes sur votre cadre théorique, mais attention à ne pas trop "préformater" la perspective de l'IA.

Si vous souhaitez également utiliser le mémo de projet pour d'autres notes qui ne doivent pas être partagées avec l'IA, vous pouvez insérer le tag "#####" dans le texte. Tout ce qui se trouve en dessous de cette marque ne sera pas envoyé à l'IA (cela fonctionne avec n'importe quel mémo dans QualCoder).

Si vous ne souhaitez pas que le mémo de projet soit envoyé à l'IA du tout, désactivez cette option dans la boîte de dialogue des paramètres.

**>> Étape suivante : [Importer des fichiers (documents empiriques)](https://qualcoder-org.github.io/doc/fr/3.2.-Files)** dans votre projet.
