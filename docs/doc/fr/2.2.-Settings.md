# Paramètres

La boîte de dialogue des paramètres permet de modifier plusieurs fonctionnalités, notamment la langue, le nom du codeur, le type et la taille de la police, ainsi que les options d'intégration de l'IA.

Plusieurs codeurs peuvent coder le même texte. Un nouveau nom de codeur peut être saisi dans la zone de texte. Vous pouvez également choisir un codeur existant dans la liste déroulante. Le nom du codeur est défini par défaut sur « default » lors de la première utilisation de QualCoder. Lors de l'ouverture d'un autre projet existant, assurez-vous que le nom du codeur est bien défini sur le nom que vous souhaitez utiliser en vérifiant et en mettant à jour les paramètres avant de travailler sur votre projet.

La langue par défaut de QualCoder est l'anglais. Actuellement, le français, l'allemand, l'italien, le portugais et l'espagnol sont disponibles comme langues alternatives. Ces traductions n'ont pas été entièrement vérifiées pour leur exactitude. Lorsque vous changez de langue, vous devez fermer puis rouvrir QualCoder pour que le changement prenne effet.

En cochant la case **Afficher les identifiants**, les identifiants numériques des codes, fichiers, cas, etc., s'affichent. Ces identifiants peuvent être utiles lors de la rédaction de requêtes SQL.

Quelques options permettent de modifier l'apparence générale de QualCoder.

Pour la transcription audio et vidéo, le format de temps et le type de crochets peuvent être sélectionnés.

Les fichiers texte très volumineux peuvent ralentir le mode de codage dans QualCoder. Une option de clic droit dans **Coder le texte** permet de charger des portions plus petites du texte. La taille par défaut est de 50 000 caractères, mais celle-ci peut être réduite dans les paramètres pour les ordinateurs plus lents.

### Sauvegardes

La sauvegarde du projet peut être configurée pour s'effectuer automatiquement toutes les heures, ou pas du tout. QualCoder conserve les sauvegardes les plus récentes avec un horodatage de date et d'heure. Le nom de la sauvegarde est : _nomduprojet_BKP_aaaammjj_hh.qda_, où _hh_ correspond à l'heure en format 24 heures. Cela signifie également qu'aucune sauvegarde supplémentaire n'est effectuée dans la même heure. Un nombre limité de sauvegardes est conservé, **les sauvegardes les plus anciennes sont supprimées**. L'option de paramétrage permet de choisir entre 1 et 10 sauvegardes.

Il est recommandé de faire une sauvegarde avant d'effectuer des modifications importantes, comme la réorganisation des codes et des catégories, la fusion de projets ou de codes. QualCoder effectue des sauvegardes horaires (en conservant jusqu'à cinq sauvegardes récentes) si cette option est activée dans les paramètres.

Renommez ou déplacez vers un autre emplacement les sauvegardes spécifiques que vous souhaitez conserver. Lorsque les sauvegardes sont activées, une option supplémentaire permet de choisir de sauvegarder également les fichiers audio/vidéo. Pour des sauvegardes complètes, il est recommandé de sauvegarder les fichiers audio et vidéo, bien que cela puisse ralentir l'ouverture du logiciel si vous avez de nombreux fichiers volumineux. Alternativement, vous pouvez lier les fichiers audio/vidéo volumineux plutôt que de les importer dans le dossier du projet. Le répertoire par défaut du projet est celui vers lequel de nombreuses opérations de fichiers pointeront par défaut, pour l'export ou l'import de fichiers.

Dans votre dossier personnel, QualCoder créera un dossier `.qualcoder` contenant un fichier `QualCoder.log` pour enregistrer les événements et un fichier `config.ini` qui contiendra de nombreux paramètres, tels que le nom du codeur actuel, les polices préférées, la langue et un répertoire de travail préféré. Ne stockez aucune donnée ou projet dans le dossier `.qualcoder`. Le seul fichier que vous pouvez y placer est un fichier `stopwords.txt`, uniquement si vous souhaitez appliquer un ensemble différent de mots vides pour les fonctions de nuage de mots et de comptage de mots. Ce dossier est réservé à l'usage interne de QualCoder pendant son exécution.
