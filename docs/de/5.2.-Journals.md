# Tagebücher

In Tagebüchern (journals) können Gedanken während des Codierens der Daten festgehalten werden. Diese dienen der Entwicklung von Ideen und Themen. 
Tagebücher können als einzelnes Tagebuch exportiert werden, oder alle Tagebuchtexte können zusammengefasst und als eine einzige Datei exportiert werden.
Tagebücher können gelöscht werden und können Attribute besitzen.

Es gibt eine Suchoption, um den Tagebuchtext zu durchsuchen. Die Suche nutzt reguläre Ausdrücke (`Regex`), um eine größere Flexibilität zu ermöglichen.

* Ein Punkt `.` dient als Platzhalter, z. B. `.aus` findet `Haus`, `Maus` und `Aus`.
* Ein `?` nach einem Zeichen findet dieses null- oder einmal, z. B. `Hunde?` findet sowohl `Hund` als auch `Hunde`.
* Ein `*` nach einem Zeichen findet dieses null- oder beliebig oft. 
* `\.` findet das Punkt-Symbol, `\?` das Fragezeichen, `\n` das Zeilenende-Symbol.

!!! tip
    [Regex-Cheat-Sheet](https://www.rexegg.com/regex-quickstart.html).

Tagebücher können nach Name oder Datum sortiert werden. Zudem können Attribute zu Tagebüchern hinzugefügt werden. Über die Rechtsklick-Funktion können Zeilen von Tagebüchern angezeigt werden, die einem bestimmten Textnamen, einem Coder oder einem Attributwert entsprechen.

Das Tagebuch-Fenster ist frei schwebend, sodass Notizen während des Codierens oder der Erstellung von Berichten eingetragen werden können.

## Tagebuchtext mit Markdown hervorheben

Einige Markdown-Elemente stehen zur Verfügung, um Formulierungen in einem Tagebuch hervorzuheben:
Verwenden Sie ein bis drei Raute-Symbole (`#`) für die Überschriftenebenen. Einzelne Sternchen (`*`) um Wörter erzeugen kursiven Text. Doppelte Sternchen (`**`) um einen Text erzeugen fettgedruckten Text.
